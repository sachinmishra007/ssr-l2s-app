What is Micro Frontend?

The concept of Micro Frontend Architecture is new thought about a web application as a composition of features owned by different independent teams. Each team having a distinct area of business that it specializes in.
  • Micro frontend is new terminology which has been introduced to decouple the monolithic frontend web application code as well as development life cycle to isolated micro apps.
  • The Micro Frontend design pattern has become a part of the application development life cycle, is often developed by a separate team, grows and gets more difficult to maintain, this type of application is called Frontend Monolith.
  • The concept of micro frontend has been brought up in order to overcome issue and limitation related to monolithic apps.
  • Micro Frontend is a Microservice Testing approach to front-end web development. The way is to build a powerful and feature-rich web application which resides on top of a Microservice architecture.
  • The architecture serves the purpose of dividing the entire technical stack based on business domain. The architecture gives you the freedom to develop the application with consistent level of flexibility, testability, and velocity that backend teams get during Microservices. Development.

How does Micro Frontend work?

The advantage of the micro front end app is to enable the cross functional or remotely located team to develop the E2E features independently. It helps to them maintain the code deploy the code without affecting other application.
  • Technology Agnostic
    Each of the team should able to choose and upgrade their stack without coordinating with other team. The architecture enables us to select the own technology and implement with considering other factors. It also help us to integrate the multiple cloud cdn hosted project under the one roof like AWS,Azure and GCP.
  • Isolate Team Code
    Build independent apps that are self contained. Don’t rely on shared state or global variables. This helps to team isolate and develop the code in different environment with different methodology.
  • Separated repositories
    The microfrontend helps to create the separate plugin also give the flexibility to do independent code check-in as well as code management.
  • Independent deployments
    On top of it allows to create the separate repositories because of independent behaviour. We can create our own CI/CD release cycle of the each of the microfrontend with any dependencies.
  • Faster builds and releases
    Since the code has only the specific business use-case that helps to create faster build and release process.
  • Autonomous teams
    The terminology of microfrontend helps us to achive the development interpedently.
  • Easier debugging and maintenance
    Since the each of the microfrontend application hosted in the independent environment. It helps to debugging the application easily because of independent nature as well as easier to maintain.

You could break the entire application based upon the specific business use case and area.

  • Composition App - It will combine all the micro apps and make SPA.
  • Header Navigation - It will be responsible for the header navigation.
  • Left Navigation - It will be responsible for the left navigation.
  • Landing Page Micro App 1 - It will hold the landing page code

How to do ?

There are various ways to achieve the microfrontend architecture. And till now there hasn’t been any standards defined to achieve the objective. But We can use below trick and techniques to achieve the objective.

  • iframe
    An iFrame is a frame within a frame. It is a component of an HTML element that allows you to embed documents, videos, and interactive media within a page. By doing this, you can display a secondary webpage on your main page.
  • Web Components

    Web components are a set of web platform APIs that allow you to create new custom, reusable, encapsulated HTML tags to use in web pages and web apps. Custom components and widgets build on the Web Component standards, will work across modern browsers, and can be used with any JavaScript library or framework that works with HTML.

    Web components are based on existing web standards. Features to support web components are currently being added to the HTML and DOM specs, letting web developers easily extend HTML with new elements with encapsulated styling and custom behavior.

  • Frint
    Single-spa is a framework for bringing together multiple JavaScript microfrontends in a frontend application. Architecting your frontend using single-spa enables many benefits, such as:
    • Gives your applications a structure
    • Environment agnostic (runs in browser, server, and CLI)
    • Modular architecture with Apps
    • Progressive and easy to adopt in existing applications
  • Single SPA
    Single-spa is a framework for bringing together multiple JavaScript microfrontends in a frontend application. Architecting your frontend using single-spa enables many benefits, such as:
    • Use multiple frameworks on the same page without page refreshing (React, AngularJS, Angular, Ember, or whatever you're using)
    • Deploy your microfrontends independently
    • Write code using a new framework, without rewriting your existing app
    • Lazy load code for improved initial load time
  • Reverse Proxy
    A reverse proxy server is a type of proxy server that typically sits behind the firewall in a private network and directs client requests to the appropriate backend server. A reverse proxy provides an additional level of abstraction and control to ensure the smooth flow of network traffic between clients and servers.
  • API Gateway
    An API gateway is an API management tool that sits between a client and a collection of backend services. An API gateway acts as a reverse proxy to accept all application programming interface (API) calls, aggregate the various services required to fulfill them, and return the appropriate result.
